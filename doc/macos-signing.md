# macOS 署名とアクセシビリティ許可（TCC）

## 問題

アドホック署名（Developer ID 証明書なしのビルド）では、再ビルドのたびにバイナリの署名が変わる。
macOS はアクセシビリティ許可（TCC）を署名に紐付けて管理するため、再ビルドすると既存の許可が無効化され、
`uiohook-napi` によるグローバル入力フック（キー打鍵・マウス検知）が動かなくなる。

## 再ビルド後の対処（暫定）

```bash
tccutil reset Accessibility com.typebloom.app
```

その後、アプリを起動し直し、
「システム設定 > プライバシーとセキュリティ > アクセシビリティ」で再度許可する。

## 恒久対応

Apple Developer Program（$99/年）に加入し、Developer ID 証明書で署名する。
署名が固定されれば、再ビルドしても TCC 許可が維持される。
